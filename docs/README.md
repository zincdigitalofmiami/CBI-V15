# Documentation

## Purpose
All project documentation - architecture, operations, feature specs.

## Structure
```
docs/
├── architecture/    # System design, data flows, decisions
├── features/        # Feature specifications and research
└── ops/             # Operational guides, runbooks
```

## What Belongs Here
- Markdown documentation (`.md`)
- Architecture diagrams
- Runbooks
- Research notes

## What Does NOT Belong Here
- Code (→ `src/`)
- Configuration (→ `config/`)
- SQL (→ `database/`)

## Key Documents
- `architecture/` - System design decisions
- `features/` - Feature engineering research and specs
- `ops/` - How to deploy, monitor, troubleshoot
